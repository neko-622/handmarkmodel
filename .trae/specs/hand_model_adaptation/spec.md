# 手部模型适配 - 产品需求文档

## 概述
- **Summary**: 修改demo_face.cpp文件，使其适配手部标注模型，将原有的人脸检测功能替换为手部检测和标注功能。
- **Purpose**: 利用现有的cut_model.onnx模型（由hand_landmarks_detector.tflite转换而来）实现手部检测和标注功能。
- **Target Users**: 开发人员和测试人员，用于测试手部检测模型的性能和准确性。

## 背景 & 上下文
- 仓库中存在一个面部识别模型的切分实现，主要用CPU完成后处理。
- 已有hand_landmarks_detector.tflite模型，已转换为cut_model.onnx模型。
- 需要修改demo_face.cpp文件，使其适配手部模型，替换原有的人脸检测功能。

## 目标
- 修改demo_face.cpp文件，使其能够加载和使用cut_model.onnx手部模型。
- 实现手部检测和标注功能，替代原有的人脸检测功能。
- 确保修改后的代码能够正常编译和运行。

## 非目标（范围外）
- 不修改模型文件本身，仅修改代码以适配现有模型。
- 不添加新的功能，仅替换现有的人脸检测功能为手部检测功能。
- 不修改其他文件，仅修改demo_face.cpp文件。

## 功能需求
- **FR-1**: 加载cut_model.onnx手部模型，替代原有的人脸检测模型。
- **FR-2**: 实现手部检测功能，包括预处理、推理和后处理步骤。
- **FR-3**: 实现手部关键点标注功能，在检测到的手部位置绘制关键点。
- **FR-4**: 保持原有的OSD可视化功能，用于显示检测结果和标注。

## 非功能需求
- **NFR-1**: 性能要求：检测和标注过程应实时运行，不出现明显卡顿。
- **NFR-2**: 兼容性：修改后的代码应与现有的硬件和软件环境兼容。
- **NFR-3**: 可维护性：代码应保持清晰、注释充分，便于后续维护和扩展。

## 约束
- **技术**: 代码应使用C++语言，遵循现有的代码风格和结构。
- **依赖**: 依赖现有的SSNE库和相关工具类。
- **模型**: 使用现有的cut_model.onnx手部模型，不进行模型修改。

## 假设
- 假设cut_model.onnx模型的输入和输出格式已知，或可以通过netron.app分析获取。
- 假设现有的SSNE库和工具类可以用于手部模型的加载和推理。
- 假设现有的OSD可视化功能可以用于显示手部检测和标注结果。

## 验收标准

### AC-1: 模型加载
- **Given**: 代码修改完成后
- **When**: 运行程序时
- **Then**: 程序应成功加载cut_model.onnx模型，无错误信息
- **Verification**: `programmatic`

### AC-2: 手部检测
- **Given**: 模型加载成功后
- **When**: 输入包含手部的图像
- **Then**: 程序应成功检测到手部位置，输出检测框
- **Verification**: `programmatic`

### AC-3: 手部关键点标注
- **Given**: 检测到手部后
- **When**: 进行关键点标注
- **Then**: 程序应成功标注手部关键点，输出关键点坐标
- **Verification**: `programmatic`

### AC-4: 可视化
- **Given**: 检测和标注完成后
- **When**: 显示结果
- **Then**: 程序应在OSD上显示手部检测框和关键点标注
- **Verification**: `human-judgment`

### AC-5: 性能
- **Given**: 程序运行时
- **When**: 处理连续帧
- **Then**: 程序应实时运行，无明显卡顿
- **Verification**: `human-judgment`
